<div align=center>
	<h2>Convention</h2>

### Branch Naming Convention

| 머릿말  | 설명                               |
| ------- | ------------------------------- |
| main    | 서비스 브랜치                      |
| develop | 배포 전 작업 기준                  |
| feature | 기능 단위 구현                    |
| hotfix  | 서비스 중 긴급 수정 건에 대한 처리     |

### Commit Convention 

| 머릿말   | 설명                                                 |
| -------- | ---------------------------------------------------- |
| feat     | 기능 구현, 추가                                      |
| fix      | 코드 수정                                           |
| design   | UI 디자인                                            |
| setting  | 패키지 설치, 개발 설정                               |
| refactor | 코드 리팩터링                                        |
| rename   | 파일명(or 폴더명)을 수정한 경우                      |
| docs     | README.md 작성 및 변경                               |
| hotfix   | 치명적인 버그를 급하게 수정하는 경우                 |

</div>
